# 배달앱 기말 프로젝트 작업 목록

마감: 2026년 6월 25일 23:59 (KST)

목표는 필수 기능을 공개 URL에서 안정적으로 시연하고, DB 설계의 이유와 실제 버그 해결 과정을 본인의 말로 설명할 수 있는 상태를 만드는 것이다.

## 기술 결정

- Next.js App Router, TypeScript, Tailwind CSS
- PostgreSQL: 로컬 Docker, 운영 Neon
- ORM 및 마이그레이션: Drizzle ORM / Drizzle Kit
- 인증: 이메일·비밀번호, bcrypt 해시, DB 세션, HTTP-only 쿠키
- 배포: GitHub → Vercel
- 패키지 관리자: pnpm
- 자동 검증: ESLint, TypeScript, Vitest, Playwright

## 완료 기준

- [ ] 공개 URL에서 회원가입 → 로그인 → 식당·메뉴 → 장바구니 → 주문 → 주문내역 → 로그아웃을 편집 없이 완료할 수 있다.
- [ ] 주문과 주문 상세가 하나의 트랜잭션으로 저장된다.
- [ ] 다른 사용자의 주문을 URL 조작으로 볼 수 없다.
- [ ] 새 데이터베이스에도 마이그레이션과 시드 명령으로 동일한 환경을 만들 수 있다.
- [ ] 모바일과 데스크톱에서 깨지는 화면이 없다.
- [ ] 새로고침, 중복 클릭, 빈 데이터, 잘못된 입력 상황을 처리한다.

## 개발 중 지킬 규칙

- 기능 하나가 완성될 때마다 작고 설명 가능한 커밋을 남긴다.
- 실제로 겪지 않은 버그를 제출용으로 꾸며내지 않는다.
- 기능 추가보다 필수 시연 흐름의 안정성을 우선한다.
- DB나 인증 오류를 임시 하드코딩 데이터로 감추지 않는다.
- 운영 배포는 마지막 날까지 미루지 않는다.
- 각 단계가 끝날 때 공개 URL 또는 프로덕션 빌드에서 다시 확인한다.
