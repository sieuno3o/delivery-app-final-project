# 동네한입 기말 프로젝트 최종 보고서

- 운영 URL: https://dongne-hanip-kohl.vercel.app
- GitHub: https://github.com/sieuno3o/delivery-app-final-project
- 기술: Next.js App Router, TypeScript, PostgreSQL/Neon, Drizzle ORM, Tailwind CSS, Vercel

## 1. 데이터베이스 구조

| 테이블 | 한 줄 설명 |
| --- | --- |
| `users` | 이메일, 비밀번호 해시, 고객·관리자 역할을 저장한다. |
| `sessions` | 원본 대신 해시한 로그인 토큰과 만료 시각을 저장한다. |
| `restaurants` | 식당 정보, 카테고리, 배달비, 최소 주문 금액을 저장한다. |
| `menu_items` | 식당별 현재 메뉴, 가격, 품절 여부를 저장한다. |
| `orders` | 주문자·식당·배송지·현재 상태·총액·중복 방지 키를 저장한다. |
| `order_items` | 주문 당시 메뉴명·단가 스냅샷과 수량을 저장한다. |
| `order_status_history` | 주문 상태가 언제 누구에 의해 변경됐는지 누적한다. |

### 왜 테이블을 나눴는가

- `orders` 1건에는 메뉴가 여러 개이므로 반복되는 메뉴 데이터는 `order_items`로 분리했다.
- `order_items`에 주문 당시 메뉴명과 단가를 복사해, 메뉴가 수정·삭제되어도 과거 주문은 바뀌지 않게 했다.
- 현재 상태는 목록 조회가 빠른 `orders.status`에 두고, 모든 변경 과정은 `order_status_history`에 보존했다.
- 계정과 세션을 분리해 한 사용자의 여러 기기 로그인을 허용하고, DB에는 세션 토큰 원문 대신 해시만 저장했다.

### 주문 저장 순서

로그인·권한 확인 → 식당과 메뉴 재조회 → 현재 가격·품절·최소 주문 금액 검증 → 서버에서 합계 재계산 → 한 트랜잭션에서 `orders`, `order_items`, 최초 상태 이력 저장 → 성공 시 커밋, 한 단계라도 실패하면 전체 롤백한다. UUID 중복 방지 키의 고유 제약으로 버튼을 여러 번 눌러도 주문은 한 건만 남는다.

## 2. 직접 겪고 해결한 문제 3개

### BUG-004 — 검은 링크 버튼의 글자가 보이지 않음 (영상 설명 버그)

- 증상: 회원가입·식당 둘러보기 버튼의 배경과 글자가 모두 검게 계산됐다.
- 원인: 레이어 밖 전역 `a { color: inherit }`가 Tailwind `@layer utilities`의 `text-white`보다 캐스케이드에서 우선했다.
- 해결/검증: 불필요한 전역 `a` 규칙을 제거했다. `getComputedStyle`의 글자색이 `rgb(33,31,28)`에서 `rgb(255,255,255)`로 바뀌고 실제 화면에서도 글자가 보이는지 확인했다.

### BUG-005 — 주문 완료 안내가 즉시 사라짐

- 증상: 주문 저장은 됐지만 `주문이 접수됐어요!` 안내가 거의 즉시 일반 상세 화면으로 바뀌었다.
- 원인: 장바구니를 비운 클라이언트 효과가 `router.replace`로 서버 컴포넌트를 다시 렌더링하며 `placed=1` 조건을 제거했다.
- 해결/검증: `history.replaceState`로 주소만 정리해 현재 완료 화면은 유지하고 새로고침 때만 일반 상세가 열리게 했다. 실제 주문 흐름으로 안내·주문번호·메뉴 스냅샷을 확인했다.

### BUG-006 — 운영 주문 시간이 9시간 느림

- 증상: 로컬은 한국 시간, Vercel은 같은 주문을 9시간 전으로 표시했다.
- 원인: `timestamptz`의 UTC 저장은 정상이었지만 날짜 포맷이 실행 서버의 기본 시간대에 의존했다.
- 해결/검증: 모든 주문 날짜 포맷에 `timeZone: "Asia/Seoul"`을 명시하고 운영 재배포 후 같은 시각이 한국 시간으로 표시되는지 확인했다.

## 3. AI 활용의 한계와 남은 제약

AI의 코드 제안은 그대로 신뢰하지 않고 계산 단위 테스트, 28개 Vitest, TypeScript·ESLint·프로덕션 빌드, Playwright 전체 흐름으로 다시 검증했다. 실제 결제·이메일 인증·실시간 배달 위치는 외부 서비스와 보안·비용 범위 때문에 시연용 주문 저장과 상태 타임라인으로 제한했다. 물리 기기의 모바일 네트워크 확인, 10분 영상 녹화와 제출은 자동화할 수 없어 마지막에 직접 수행해야 한다.
